## Что это

Demo-ready версия KnowledgeOS. Все внешние зависимости (Stripe, Gemini, email) работают через mock providers — платформу можно запустить и показать без единого API ключа.

Эволюция от v1.0.0 (10 сервисов, 1343 теста) до полностью демонстрируемой платформы (12 сервисов, 1402 теста, 28 страниц).

## Что нового (diff от v1.0.0)

### Mock Mode — полностью автономная работа

| Зависимость | Env Var | Без ключа | Mock провайдер |
|-------------|---------|-----------|---------------|
| Gemini LLM | `GEMINI_API_KEY` | AI возвращает реалистичные mock-ответы | `MockLLMProvider` |
| Stripe | `STRIPE_SECRET_KEY` | Платежи возвращают fake success | `MockStripeClient` |
| Resend email | `RESEND_API_KEY` | Emails логируются, не отправляются | `StubEmailClient` |
| Gemini Embeddings | `GEMINI_API_KEY` | Случайные векторы (поиск degraded) | `StubEmbeddingClient` |

### Новые Rust сервисы

| Service | Port | Назначение |
|---------|------|-----------|
| ws-gateway | 8011 | WebSocket real-time нотификации |
| embedding-orchestrator | 8009 | Concurrent embedding API proxy |

### Onboarding Flow (B2B)

```
Регистрация → /onboarding (6 шагов) → /dashboard
```

- `OnboardingGuard` — client-side redirect guard (localStorage)
- 6-step wizard: Organization → Profile → Experience → Assessment → Plan → Start
- Middleware разрешает `/onboarding` без auth
- При завершении — `onboarding_complete` в localStorage

## Demo Walkthrough (5 минут)

### Quick Start
```bash
cp .env.example .env
docker compose -f docker-compose.dev.yml up -d --build
docker compose -f docker-compose.dev.yml --profile seed up seed
cd apps/buyer && pnpm install && pnpm dev
# Open http://localhost:3001
# Login: demo@acme.com / demo123
```

### 1. Dashboard (1 мин)
- Персонализированный дашборд: mission card, streaks, trust level
- Team progress (для tech leads)
- Command palette (Cmd+K)

### 2. Knowledge Graph (1 мин)
- `/graph` — React Flow с concept nodes
- Mastery levels по цветам
- Click → Concept Hub

### 3. AI Mission (1 мин)
- Start mission → tri-agent coaching
- Strategist → Designer → Coach
- XP и badge по завершению

### 4. Smart Search (1 мин)
- `/search` — внутренний KB + external
- Query router: RAG vs Gemini

### 5. Flashcards & Gamification (1 мин)
- `/flashcards` — FSRS spaced repetition
- `/badges` — XP, streaks, badges
- `/missions` — список активных/завершённых

## Известные ограничения

| Ограничение | Влияние | Приоритет |
|-------------|---------|-----------|
| RAG embeddings — random vectors | Поиск возвращает нерелевантные результаты без GEMINI_API_KEY | Low (demo) |
| AI credits в Redis | Не засеиваются через SQL | Low |
| Buyer app не в docker-compose | Запуск `pnpm dev` отдельно | Low |
| Нет certificates / pretests data | Пустые секции | Low |
| 6 pre-existing test failures | 3 enrollment + 3 notification (мисматч test ↔ implementation) | Low |

## Честная оценка

**Что получилось хорошо:**
- Полностью автономная демо: `docker compose up` + `pnpm dev` = рабочая платформа без API ключей
- 1402 теста — надёжная safety net
- 28 страниц buyer app — каждая с loading/error/empty states
- Dark Knowledge UI — cohesive, polished, не generic
- 4 mock провайдера — graceful degradation вместо crash
- Детерминистичный seed — воспроизводимая демо-среда
- Tri-agent AI coaching — уникальный UX для B2B onboarding

**Что сыро:**
- RAG search без реальных embeddings — нерелевантные результаты
- Нет CI/CD — ручное тестирование
- Seller app — не обновлён под B2B
- Нет E2E тестов (Playwright)
- 6 pre-existing failures в enrollment и notification

**Demo readiness: 8.5/10**
